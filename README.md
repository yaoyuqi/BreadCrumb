## Review

For easier review, you can check the steps through the Git commits.

BreadCrumb is a small library management exercise with two projects:

- `Library`: ASP.NET Core backend API
- `LibReact`: React + Vite frontend

The backend stores and serves book data. The frontend shows the library UI and calls the backend API through the Vite dev proxy.

## Project Structure

```text
BreadCrumb/
├─ Library/    # ASP.NET Core Web API
└─ LibReact/   # React frontend
```

## Backend: `Library`

The backend is an ASP.NET Core Web API targeting `.NET 10` and using:

- Entity Framework Core
- SQLite
- Swagger / OpenAPI

### Backend responsibilities

- return paged book data
- filter books by title
- add a new book
- borrow a book
- return a book
- apply database migrations on startup
- seed sample book data on startup when the database is empty

### Main API endpoints

- `GET /api/books`
- `POST /api/books`
- `PUT /api/books/{id}`

### Local backend URLs

From the current launch settings:

- `http://localhost:5144`
- `https://localhost:7228`

Swagger UI is available in development mode.

### Run the backend

```powershell
cd Library\Library
dotnet run
```

## Frontend: `LibReact`

The frontend is a React app built with:

- React
- TypeScript
- Vite
- React Bootstrap
- Bootstrap

### Frontend responsibilities

- display the library page
- search books
- paginate results
- trigger borrow / return actions
- show the add-book modal

### Current frontend limitations

- the add-book modal is present, but submit is not connected to the backend yet
- the delete button is present, but delete behavior is not implemented yet

### Run the frontend

```powershell
cd LibReact
npm install
npm run dev
```

Vite usually starts on:

- `http://localhost:5173`

## Frontend to Backend Connection

The React app uses the Vite development proxy for API calls.

Current proxy target:

- `/api` -> `https://localhost:7228`

Example frontend request:

```text
/api/books?page=1&pageSize=10
```

This is forwarded to the backend during development.

## Typical Development Flow

Start the backend first:

```powershell
cd Library\Library
dotnet run
```

Then start the frontend:

```powershell
cd LibReact
npm run dev
```

## Notes

- The backend seeds more than 20 sample books when the database is empty.
- Swagger is useful for checking backend responses directly while developing the frontend.
- The frontend currently focuses on the core library workflow rather than full CRUD completion.
